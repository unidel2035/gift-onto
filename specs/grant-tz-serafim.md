## 3. Цели и задачи

### 3.1. Цель ОР

Создание демонстрационного образца (ДО) **программно-аппаратного модуля автономного роевого интеллекта** (далее — Модуль), обеспечивающего:

- Автономное (без участия оператора) тактическое поведение БпЛА в составе роя
- Стратегическое планирование и выработку инструментов на наземной станции
- Функционирование роя **без интернет-соединения и сигналов ГНСС**
- Интеграцию с существующими БпЛА через стандартные интерфейсы (MAVLink, UART, SPI)

**Модуль не включает планер БпЛА, двигатели и полезную нагрузку.**
Модуль является комплектом бортового вычислителя с предустановленным ПО и наземного ПО, устанавливаемым на БпЛА заказчика или на стандартные рамы (типоразмер 7-10 дюймов / БЛА самолётного типа класса «крыло» до 2 кг).

### 3.2. Базовые принципы

| № | Принцип | Реализация в Модуле |
|---|---------|---------------------|
| 1 | Автономное управление единицами | **Serafim V2 Q8** — квантованная LLM (1.6 GB) на борту. 5/5 точных тактических решений. |
| 2 | Самоорганизация роя | **UWB-сетка** (DW1000, ±10 см, пассивная) + **аукцион целей** без конфликтов |
| 3 | Минимизация связи | **Wavelet-сжатие** (×8-12) + бинарный протокол. LoRa-пакеты ≤51 байт. |
| 4 | Масштабируемость | Mesh-архитектура: каждый новый агент = новый узел сети. До 10+ БпЛА. |
| 5 | Гетерогенные рои | **HAL**: автоопределение Nvidia/Rockchip/Hailo/Intel/CPU на любом борту |
| 6 | Машинное обучение | YOLOv8 (14 классов, mAP 0.995) + Serafim (5/5 правильных решений) |
| 7 | Распределённые вычисления | Три уровня: борт (200ms) → ретранслятор (автономия) → земля (стратегия, 5-40s) |
| 8 | Мультисенсорная интеграция | Камера + UWB + IMU + барометр + LoRa RSSI. Поддержка сторонних сенсоров. |
| 9 | Интеграция стороннего оборудования | HAL: автообнаружение + стандартные интерфейсы (MAVLink, UART, SPI, I2C) |
| 10 | Без интернета | LoRa mesh (868 МГц, 10+ км) + опциональный LTE-ретранслятор как мост |

### 3.3. Трёхуровневая ИИ-архитектура (уникальное преимущество)

```
УРОВЕНЬ 1 — БОРТОВОЙ ИИ (каждый БпЛА)
  Модель:        Serafim V2 Q8 (1.6 GB, квантованная LLM)
  Железо:        Orange Pi 5 (8 GB RAM, Rockchip NPU 6 TOPS)
  Решения:       ATTACK / OBSERVE / RTB / PATROL
  Задержка:      ≤200 мс (на борту, без связи)
  Трафик:        0 (полностью локальный инференс)
  Связь потеряна → продолжает миссию автономно

УРОВЕНЬ 2 — НАЗЕМНЫЙ ИИ (станция оператора)
  Thinker:       DeepSeek-R1 8B — стратегический анализ, приоритеты целей, тактика роя
  Coder:         DeepSeek-Coder V2 — написание инструментов для роя во время миссии
  Железо:        Ноутбук с GPU 16GB / Intel Arc A770 / Б/у RTX 3060
  Задержка:      5-40 с (зависит от GPU)
  Трафик к рою:  Сжатые LoRa-пакеты ≤51 байт через ретранслятор

УРОВЕНЬ 3 — ВОЗДУШНЫЙ РЕТРАНСЛЯТОР (1 БпЛА-крыло)
  Мост:          Mesh-сеть роя (LoRa 868 МГц) ↔ интернет-каналы (LTE/Гонец/Sprint-030)
  Дублер:        Serafim на борту — если связь с землёй потеряна, ведёт рой автономно
  Высота:        300-500 м (прямая видимость LTE-вышек на 50-100 км)
  Время:         45-60 мин (крыло, Li-Ion 10000mAh)
```

## 4. Технические требования к ДО

### 4.1. Назначение

ДО предназначен для оснащения существующих БпЛА (мультироторных и самолётного типа) модулем автономного роевого интеллекта, обеспечивающим:

- Поражение наземных объектов тактическими подразделениями
- Дневное и ночное применение
- Скорость ветра у земли до 10 м/с
- Температура от −20°С до +50°С
- Повышенная влажность до 98% (при +3°С)
- **Отсутствие сигналов ГНСС** (GPS-отрицаемая среда)
- **Отсутствие интернет-соединения** (автономный режим)

|-----------|--------|------------|

| Программная документация | 1 комплект | Исходные тексты, описание, сборка в VirtualBox |

### 4.3. Функциональные требования

**Требования к бортовому ИИ (Serafim):**
- Принятие тактических решений за ≤200 мс без связи с оператором
- Действия: ATTACK (атака), OBSERVE (наблюдение), RTB (возврат), PATROL (патруль)
- Автоматический RTB при батарее <15%
- Переключение в одиночный режим при потере связи с роем
- Обратное переключение в роевой режим при восстановлении связи

**Требования к наземному ИИ (Thinker + Coder):**
- Thinker: стратегический анализ за ≤10 с (GPU) / ≤60 с (CPU)
- Coder: написание инструментов для роя за ≤5 с (GPU) / ≤45 с (CPU)
- Поддержка сжатого канала связи (LoRa ≤51 байт/пакет)
- Авто-выбор интернет-канала: LTE → Sprint-030 → Гонец → автономный режим

**Требования к взаимному позиционированию (UWB):**
- Пассивное измерение дистанций (не излучает — не пеленгуется)
- Точность ±10 см на дистанции до 500 м
- Трилатерация при 3+ соседях → относительные координаты
- 1+ БпЛА с известной позицией → абсолютные координаты всего роя

**Требования к связи (LoRa mesh):**
- Дальность БпЛА-БпЛА: до 5 км (прямая), до 50 км (mesh-ретрансляция)
- Дальность БпЛА-ретранслятор: до 10 км
- Частота: 868 МГц (помехоустойчивая)
- Размер пакета: ≤51 байт (сжатый бинарный протокол)
- Wavelet-сжатие телеметрии: ×8-12 экономии полосы

**Требования к обнаружению целей (YOLO):**
- 14 классов: свои дроны (6 типов) + наземные цели (8 типов)
- Дальность обнаружения: до 1500 м (зависит от камеры носителя)
- mAP50: ≥0.95 на тестовой выборке

**Требования безопасности:**
- Ограничение поиска целей вблизи точки пуска
- Проверка допустимости параметров полётного задания
- Запрещённые для полёта и поиска зоны
- Предотвращение столкновений (UWB + IMU + terrain avoidance)

## 5. Отчётная документация

Исполнитель разрабатывает:

1. **Проектная КД**: чертежи общего вида бортового вычислителя/ретранслятора/наземной станции, пояснительная записка, ведомость покупных изделий
2. **Эскизная КД**: габаритные чертежи, ПМИ, эксплуатационные документы
3. **Программная документация**: текст программы, описание, руководство системного программиста, руководство программиста, пояснительная записка (математические методы, алгоритмы, допущения)
4. Акты готовности, протоколы испытаний и апробации, акт научно-технической приёмки

Текст программы — в виде виртуальной машины VirtualBox с настроенным окружением для сборки из исходных текстов.

**Единственная новая работа:** интеграция компонентов на реальном железе и лётные испытания. Все компоненты по отдельности уже работают.
